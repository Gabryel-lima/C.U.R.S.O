## Direção inicial

| Tipo de teste | Objetivo |
|---------------|----------|
| Unitário | Validar classes de domínio e regras de negócio |
| Integração | Validar operações com MySQL |
| API | Validar endpoints CRUD e respostas HTTP |

## Ordem recomendada

1. Criar testes das entidades e regras de negócio.
2. Criar testes da camada de persistência.
3. Criar testes da API com cenários de sucesso e erro.
